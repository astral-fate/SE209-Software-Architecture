Software Architecture and Design

Instructor: Eng. Shymaa Othman
This course is designed to provide a comprehensive understanding of software architecture principles and commonly used design patterns. It will equip participants with the knowledge and skills needed to design scalable, maintainable, and efficient software systems.

Course Syllabus

Unit Main Topics Key Concepts Methodologies / Tools
Unit 1 - Software Development Life Cycle (SDLC)
- Software Architecture Fundamentals
- Software Architecture Quality Attributes
- Styles vs. Patterns vs. Design Patterns
- Monolithic vs. Distributed Architectures
- Architecture Partitioning (Technical vs. Domain) 		- SDLC Phases (Initiation, Requirements, Analysis, Design, Development, Testing, Deployment, Maintenance)
- Architecture as a blueprint
- Quality Attributes (Scalability, Reliability, Maintainability, Performance, Security, etc.)
- Monolithic Pros (Simple, low cost, faster development)
- Monolithic Cons (Challenging maintenance, limited scalability, high MTTR)
- Distributed Pros (Scalable, reliable, high fault tolerance, flexible, testable)
- Distributed Cons (Higher cost, fallacies of distributed computing)
- 8 Fallacies of Distributed Computing
- Aligning partitioning with team structure and expected changes 		- SDLC
- SRS (Software Requirement Specification)
- Architectural Styles (Monolithic, Distributed)
- Architectural Patterns (MVC, Microkernel, etc.)
- Design Patterns (Creational, Structural, Behavioral)
Unit 2 - Layered Architecture (N-Tier)
- Layer Definitions (Presentation, Business, etc.)
- Key Concept: Layers of Isolation
- Open vs. Closed Layers
- Service Layer
- Architecture Sinkhole Anti-Pattern 		- Monolithic and Technically Partitioned
- Presentation Layer: Handles user interaction, UI
- Business Layer: Executes business rules and logic
- Persistence Layer (DAO): Accesses the database (CRUD operations)
- Database Layer: Stores data, indexes, and tables
- Service Layer: Optional layer for common, shared services (logging, auditing)
- Isolation: Changes in one layer generally don't impact others
- Closed Layer: Request must pass through the layer directly below it
- Open Layer: Allows requests to bypass it and go to the layer below 		- N-Tier Architecture
- DAO (Data Access Object)
- SQL
- Sinkhole Anti-Pattern

Course Objectives

  • Provide a comprehensive understanding of software architecture principles
  • Learn commonly used design patterns
  • Gain the skills to design scalable, maintainable, and efficient software systems
  • Understand the Software Development Life Cycle (SDLC) and where architecture fits
  • Define and identify Software Architecture Quality Attributes (e.g., scalability, reliability, maintainability)
  • Differentiate between architectural styles, patterns, and design patterns
  • Analyze the pros and cons of Monolithic and Distributed architectures
  • Understand technical partitioning vs. domain partitioning
  • Gain in-depth knowledge of Layered (N-Tier) architecture
